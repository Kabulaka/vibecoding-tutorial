# CLAUDE.md - VibeCoding 教程项目

> 本项目是面向开发者的 VibeCoding 快速入门教程，采用 GitBook 格式组织。

## 项目概述

- **目标读者**：有开发经验（1-3 年初级到 5 年以上资深）但未接触 VibeCoding 的技术人员
- **学习时长**：2-2.5 小时
- **内容形式**：GitBook 文档，模块化小文件组织
- **核心目标**：让读者掌握与 AI 协作开发的能力，从"写代码的人"转变为"指挥写代码的人"

## 目录结构

```
vibecoding-tutorial/
├── README.md                    # 教程介绍
├── SUMMARY.md                   # GitBook 目录
├── quick-start/                 # 第一章：快速开始（3文件）
├── core-concepts/               # 第二章：核心范式（5文件）
├── practical-skills/            # 第三章：实战操纵（5文件）
├── case-study/                  # 第四章：完整案例（1文件）
└── best-practices/              # 第五章：最佳实践（6文件）
```

## 内容编写规范

### 文件组织
- 每个文件聚焦一个知识点，高内聚低耦合
- 文件名：小写，连字符分隔，3-5 个单词
- 单文件控制在 800-1500 字，阅读时间 5-10 分钟

### 写作风格
- 语言：简体中文，技术术语保留英文
- 语气：亲切、实用、略带幽默
- 结构：目标 → 内容 → 实践 → 小结
- 代码示例：必须可运行，注释详细

### 跨文件引用
- 使用相对路径链接：`../core-concepts/context-is-all.md`
- 关键概念首次出现时链接到对应文件
- 案例章节可以引用前面所有章节

## 关键技术点

本教程涉及的核心工具和概念：
- **Claude Code**：主要的 AI 开发工具
- **Superpowers**：Claude Code 的技能系统
- **MCP**：Model Context Protocol，外部工具连接
- **Hooks**：自动化触发机制
- **Context 管理**：上下文窗口优化策略

## 开发工作流

1. 按章节顺序编写内容
2. 每章完成后自测阅读流畅度
3. 案例章节需验证所有前置知识点已覆盖
4. 最终进行全文链接检查和排版统一
5. 需要隔离开发时，优先使用项目根目录下的 `.worktrees/` 作为 git worktree 目录

## 注意事项

- 保持内容的实用性，避免纯理论说教
- 诚实面对 AI 的局限性，不夸大能力
- 示例代码需经过实际验证
- 教程内容中的版本说明：跟随 Claude Code 版本更新关键内容
- Makefile 构建版本：统一来自 `git describe --tags --always --dirty`
- `make build`：除构建 mdBook 外，还会自动生成带版本号的压缩包
- 不在 `make` 命令行中手工指定版本号
- `make clean`：删除构建目录以及当前版本对应的压缩包

# VibeCoding 教程项目初始化设计文档

- 日期：2026-04-01
- 范围：`vibecoding-tutorial` 初始目录结构、`README.md`、`SUMMARY.md`
- 状态：已确认，可作为后续内容编写的基线

## 1. 目标

为 VibeCoding 教程项目建立第一版可用的 GitBook 骨架，使项目具备清晰的目录结构、课程首页和导航目录，便于后续分章节逐步填充内容。

本次设计只覆盖以下内容：

1. 创建教程主目录及五个章节子目录
2. 编写 `README.md` 作为教程首页
3. 编写 `SUMMARY.md` 作为 GitBook 导航入口

本次设计**不包含**：

- 各章节正文文件的创建与编写
- appendix、glossary、references、changelog 等附加内容
- 图片、示意图、自动化构建或发布配置

## 2. 设计决策

### 2.1 目录结构

目录结构严格遵循既有大纲，不做扩展，不预留附录目录。

```text
vibecoding-tutorial/
├── README.md
├── SUMMARY.md
├── quick-start/
├── core-concepts/
├── practical-skills/
├── case-study/
└── best-practices/
```

这样做的原因是：当前目标是尽快建立稳定骨架，而不是提前铺设尚未确定的扩展结构。后续如果确有附录需求，再按实际内容补充即可。

### 2.2 README 的职责边界

`README.md` 采用“课程首页”定位，而不是完整目录镜像。

它负责回答这些问题：

- 这套教程是什么
- 适合谁
- 学完能获得什么
- 整体学习路径是什么
- 应该从哪里开始阅读

它**不负责**逐条展开全部章节子项，以避免与 `SUMMARY.md` 产生重复维护成本。

### 2.3 SUMMARY 的职责边界

`SUMMARY.md` 只承担 GitBook 导航作用，严格映射大纲中的正式章节与文章条目。

要求：

- 保持层级清晰
- 路径全部使用相对路径
- 不加入大纲之外的条目
- 不在此文件中承载导读说明

## 3. 可选方案与最终选择

本次讨论过三种组织方式：

### 方案 A：README 做课程首页，SUMMARY 做纯导航

这是最终采用方案。

优点：

- 职责清晰
- 最符合 GitBook 使用习惯
- 后续维护成本最低
- README 可面向读者，SUMMARY 可面向结构

缺点：

- README 与 SUMMARY 不完全对称

### 方案 B：README 与 SUMMARY 高度镜像

未采用。

原因：信息重复，后续调整章节时容易出现双处同步问题。

### 方案 C：README 偏宣传页，SUMMARY 偏教学结构

未采用。

原因：当前项目核心用途是教程阅读，不需要把 README 做成更偏宣传性质的展示页。

## 4. README 内容结构

`README.md` 应包含以下模块：

1. 标题与一句话定位
2. 什么是 VibeCoding
3. 教程适用读者
4. 学习收益
5. 学习路径总览
6. 阅读建议
7. 起始入口

写作要求：

- 使用简体中文
- 技术术语保留英文
- 语气亲切、实用、略带幽默
- 明确强调 AI 协作开发的边界，不夸大能力
- 让第一次进入仓库的读者能在几分钟内建立整体认知

## 5. SUMMARY 内容结构

`SUMMARY.md` 应包含：

- 教程介绍
- 第一章：快速开始（3 篇）
- 第二章：核心范式（5 篇）
- 第三章：实战操纵（5 篇）
- 第四章：完整案例（1 篇）
- 第五章：最佳实践（7 篇）

所有标题、顺序、文件路径均以 `vibecoding-tutorial-outline.md` 为准。

## 6. 数据流与依赖关系

这次工作没有运行时数据流，只有内容结构依赖关系：

1. `vibecoding-tutorial-outline.md` 提供权威目录蓝图
2. `CLAUDE.md` 提供写作风格和项目约束
3. `README.md` 根据蓝图生成课程首页
4. `SUMMARY.md` 根据蓝图生成导航目录

因此，唯一的权威来源是：**大纲决定结构，CLAUDE.md 决定风格**。

## 7. 错误控制与一致性策略

为了避免后续出现文档结构漂移，本次设计明确以下约束：

- `SUMMARY.md` 中不加入未在大纲中出现的条目
- `README.md` 只做导读，不做全量目录复制
- 章节名称、文件名、顺序均以大纲为准
- 后续新增章节时，必须同时评估是否需要更新 `README.md` 的学习路径总览

## 8. 验收标准

本阶段完成后，应满足：

1. `vibecoding-tutorial/` 目录存在
2. 五个章节子目录存在
3. `README.md` 能作为教程首页单独阅读
4. `SUMMARY.md` 能作为 GitBook 目录直接使用
5. `SUMMARY.md` 中无 appendix 或其他未授权扩展内容
6. 文风符合项目 CLAUDE.md 约束

## 9. 后续工作建议

下一阶段应进入“内容填充”而非结构重构，建议按以下顺序推进：

1. 第一章 3 篇
2. 第二章 5 篇
3. 第三章 5 篇
4. 第四章案例
5. 第五章最佳实践

这样可以保持与教程学习路径一致，也方便每章完成后单独检查风格统一性和链接完整性。

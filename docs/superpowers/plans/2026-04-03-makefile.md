# Makefile Versioned Package Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修改仓库根目录 `Makefile`，让 `make build` 在构建 mdBook 后额外产出一个带 git 自动版本号的压缩包，并提供 `make version` 查看版本元数据。

**Architecture:** 保持单文件 Makefile 方案，在现有目标上增加 `PROJECT_NAME`、`VERSION`、`BUILD_TIME`、`COMMIT_HASH`、`PACKAGE_FILE` 等变量，并新增 `package` 与 `version` 目标。`build` 负责先执行 `mdbook build`，再调用 `package`，版本字符串统一来自 `git describe --tags --always --dirty`，避免构建时手工传参。

**Tech Stack:** Make, mdBook, git, tar, Python 3, POSIX shell

### Task 2: 实现自动版本变量与 `package` / `version` 目标

**Files:**
- Modify: `Makefile:1-30`

- [ ] **Step 1: 在 `Makefile` 顶部加入版本与压缩包相关变量**

Replace the variable block in `Makefile` with:
```make
.DEFAULT_GOAL := build

PROJECT_NAME := vibecoding-tutorial-book
DIST_DIR := dist/vibecoding-tutorial-book
INDEX_HTML := $(DIST_DIR)/index.html
VERSION := $(shell git describe --tags --always --dirty 2>/dev/null || echo "v0.0.0-dev")
BUILD_TIME := $(shell date -u '+%Y-%m-%d_%H:%M:%S')
COMMIT_HASH := $(shell git rev-parse --short HEAD 2>/dev/null || echo "unknown")
PACKAGE_FILE := dist/$(PROJECT_NAME)-$(VERSION).tar.gz

.PHONY: build package serve clean check open version
```

- [ ] **Step 2: 新增 `package` 目标，要求把整个 mdBook 产物目录打成带版本号压缩包**

Insert after the variable block:
```make
package:
	@test -d "$(DIST_DIR)"
	@(cd dist && tar -czf "$(PROJECT_NAME)-$(VERSION).tar.gz" "$(PROJECT_NAME)")
```

- [ ] **Step 3: 新增 `version` 目标，输出版本元数据**

Append to `Makefile`:
```make
version:
	@echo "项目: $(PROJECT_NAME)"
	@echo "版本: $(VERSION)"
	@echo "构建时间: $(BUILD_TIME)"
	@echo "提交哈希: $(COMMIT_HASH)"
```

- [ ] **Step 4: 运行 `make version`，确认新目标可用**

Run:
```bash
cd /home/nika/workspace/doc/vibecoding && make version
```

Expected: PASS，输出包含 `版本:`、`构建时间:`、`提交哈希:` 三行，且版本值不需要手工传入。
